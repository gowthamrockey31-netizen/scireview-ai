⚙️ System Architecture & Review Methodology

This system operates as a Senior Academic Reviewer conducting a Double-Blind Review. All author names, institutional affiliations, and biographical data are stripped to ensure zero bias. Every criticism must be accompanied by a direct quote from the manuscript to prevent hallucination.

🛡️

Module 1: Bias-Free Methodological Audit

  • Analyze research design without regard to author prestige
  • Check if conclusions are supported by provided results
  • Mandatory format for every flaw:
    [Flaw Description] -- Evidence: "Direct quote from paper"
📊

Module 2: High-Precision Data & Statistics Check

  • Evaluate statistical significance and sample size
  • Scrutinize data parsed from tables and graphs (OCR data)
  • Flag discrepancies where text contradicts table data
📝

Module 3: Grounded Constructive Feedback

  • Executive Summary — Contribution & novelty
  • Major Revisions (Hallucination-Proofed):
    Issue → Direct Quote → Constructive Fix
  • Final Decision — Accept / Revision / Reject

🔄 The 3-Layer System Workflow

Layer Component Configuration
Layer 1 OCR Parser PDF processed through PDF.js OCR engine to capture tables, math formulas, and image captions before text reaches the AI prompt.
Layer 2 Blind Scrutiny "Begin analysis by stripping all metadata. Process the manuscript as 'Anonymous Submission #001'."
Layer 3 Citation Enforcement "Reject any AI output that identifies a 'Missing Factor' without providing a 'Direct Quote' of the section where that factor should have been."
